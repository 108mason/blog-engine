Content Management

Posts are managed directly in GitHub — no dashboard needed.

1

Trigger a scan

Go to Actions → Scan & Generate Posts → Run workflow in your GitHub repo. New draft posts will be written to blog/data/posts.json automatically.

2

Review & edit a draft

Open blog/data/posts.json in the GitHub web editor. Find the post you want (status: "draft"), edit the title, body, or any other field directly in the JSON.

3

Publish a post

Change "status": "draft" → "status": "published" and set "publishedAt" to today's ISO date (e.g. "2025-06-01T10:00:00.000Z"). Commit the file — GitHub Pages deploys automatically.

4

Delete a draft

Remove the entire post object from the JSON array in the GitHub editor and commit.
